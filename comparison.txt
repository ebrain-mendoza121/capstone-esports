# Esports Analytics Platform — Complete Capstone Implementation Plan

## 1. Project Overview

This project builds a **web-based esports analytics platform for League of Legends** that ingests Riot API data, processes match timelines and drafts, extracts strategic and spatial features, trains predictive models, and presents insights through interactive dashboards.

The system goes beyond static statistics and provides:

* draft intelligence
* timeline-based game analysis
* team performance evaluation
* map-based macro analysis
* predictive machine learning models
* coaching-style recommendations

The project demonstrates complexity across **data engineering, backend architecture, analytics pipelines, machine learning, and visualization systems**.

---

# 2. Final System Architecture

```
Riot API
   ↓
FastAPI Ingestion Services
   ↓
PostgreSQL Raw Data Layer
   ↓
Feature Engineering & Analytics Layer
   ↓
Machine Learning Training & Inference
   ↓
REST API Endpoints
   ↓
React Dashboard & Visualization
```

# 3. Core System Modules

## Module A — Data Ingestion and Storage

This layer retrieves and normalizes data from Riot's Match-V5 API.

### Responsibilities

* Riot ID → PUUID resolution
* match ID retrieval
* match ingestion
* timeline ingestion
* draft ingestion
* backfill jobs
* coverage tracking

### Core Tables

Existing:

* `players`
* `matches`
* `participant_stats`
* `team_objectives`
* `team_bans`
* `draft_actions`
* `derived_metrics`

New timeline tables:

#### match_timeline_frames

Stores player state snapshots.

Columns:

* id
* match_id
* timestamp_ms
* participant_id
* team_id
* position_x
* position_y
* current_gold
* total_gold
* xp
* level
* minions_killed
* jungle_minions_killed

#### match_timeline_events

Stores timeline events.

Columns:

* id
* match_id
* timestamp_ms
* event_type
* participant_id
* team_id
* killer_id

* assisting_participant_ids
* item_id
* position_x

* lane_type

These tables enable advanced analytics and ML features.

# 9. Backend API Structure

## Ingestion

* POST `/ingest/player`
* POST `/backfill/derived`
* POST `/backfill/draft-actions`
* POST `/backfill/timeline`

## Coverage

* GET `/backfill/status`
* GET `/backfill/draft-actions/status`
* GET `/backfill/timeline/status`

## Draft Analytics

* GET `/draft/picks`
* GET `/draft/bans`
* GET `/draft/synergy`
* GET `/draft/counters`

## Timeline Analytics

* GET `/timeline/summary/{match_id}`
* GET `/analytics/ganks`
* GET `/analytics/teamfights`
* GET `/analytics/objectives`
* GET `/analytics/map/heatmap`

## Machine Learning

* POST `/ml/train/draft-win`
* POST `/ml/train/midgame-win`
* GET `/ml/models/status`
* POST `/ml/predict/draft-win`
* POST `/ml/predict/midgame-win`
* POST `/ml/evaluate/model`

## Debug

* GET `/debug/db-summary`
* GET `/debug/first-match-positions`
* POST `/debug/smoke-run`
